# 테트리스 (교육용)

HTML, CSS, JavaScript만으로 만드는 브라우저 테트리스 게임입니다.  
프론트엔드 입문자가 **파일 분리**, **게임 루프**, **충돌 판정**, **상태 관리**를 학습할 수 있도록 구성했습니다.

빌드 도구나 외부 라이브러리 없이 동작하며, 로컬 파일 실행과 GitHub Pages 배포를 모두 지원합니다.

## 실행 방법

### 로컬에서 실행

1. 이 저장소를 클론하거나 폴더를 연다.
2. `index.html`을 더블클릭하거나 브라우저 창에 끌어다 놓는다.
3. **시작** 버튼을 눌러 게임을 시작한다.

> 별도의 설치·빌드 과정이 필요 없습니다.

### GitHub Pages에서 실행

저장소를 GitHub에 push하고 Pages를 활성화한 뒤, 아래 URL로 접속합니다.

```
https://<GitHub사용자명>.github.io/<저장소명>/
```

## 조작법

| 키 | 동작 |
|----|------|
| `←` (ArrowLeft) | 왼쪽 이동 |
| `→` (ArrowRight) | 오른쪽 이동 |
| `↓` (ArrowDown) | 한 칸 빠르게 내리기 |
| `↑` (ArrowUp) | 회전 |
| `Space` | 즉시 낙하 (hard drop) |

- 벽이나 고정된 블록과 충돌하면 이동·회전이 적용되지 않습니다.
- 회전 후 충돌이 발생하면 회전이 취소됩니다.
- 게임 중에는 브라우저 창을 클릭해 포커스를 맞춘 뒤 키를 입력하세요.

## 구현 기능

| 기능 | 설명 |
|------|------|
| 게임 보드 | 10열 × 20행 CSS Grid |
| 테트로미노 | I, O, T, S, Z, J, L 7종 |
| 자동 낙하 | 일정 간격(800ms)으로 블록 하강 |
| 충돌 판정 | 벽·고정 블록·보드 상단 경계 검사 |
| 키보드 조작 | 이동, 회전, 소프트 드롭, 하드 드롭 |
| 줄 삭제 | 가득 찬 줄 제거 후 위 줄 하강 |
| 점수 | 삭제한 줄 수에 따라 점수 증가 |
| 게임 오버 | 스폰 불가 또는 상단 overflow 시 종료 |
| 재시작 | 보드·점수·타이머·상태 초기화 |

## 점수 규칙

| 삭제한 줄 수 | 획득 점수 |
|-------------|----------|
| 1줄 | 100 |
| 2줄 | 300 |
| 3줄 | 500 |
| 4줄 | 800 |

5줄 이상이 한 번에 삭제되면 4줄 단위(800점)와 나머지 줄 수에 따른 점수를 합산합니다.  
(예: 5줄 → 800 + 100 = 900점)

## 게임 오버 조건

- 새 블록을 스폰 위치에 둘 수 없을 때
- 블록이 보드 상단 밖으로 나간 상태에서 고정될 때

**재시작** 버튼을 누르면 보드, 점수, 타이머, 게임 상태가 초기화됩니다.

## 파일 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 화면 구조 (보드, 점수, 버튼, 조작법) |
| `style.css` | 레이아웃과 디자인 |
| `script.js` | 게임 로직 |
| `README.md` | 프로젝트 문서 |

## 품질 점검 방법

배포 전·후 아래 항목을 수동으로 확인하세요.

| # | 확인 항목 | 기대 결과 |
|---|----------|-----------|
| 1 | 페이지 로드 | 빈 보드, 점수 0, **시작** 버튼 표시 |
| 2 | 콘솔 | 개발자 도구(F12)에 에러 없음 |
| 3 | 게임 시작 | **시작** 후 블록 낙하 시작 |
| 4 | 이동·회전 | ← → ↑ 키 정상 동작, 벽·블록 충돌 시 멈춤 |
| 5 | 낙하 | ↓ 소프트 드롭, Space 하드 드롭 동작 |
| 6 | 줄 삭제 | 가로 한 줄 완성 시 삭제·점수 증가 |
| 7 | 게임 오버 | 스택이 꽉 차면 메시지 표시, 입력 중지 |
| 8 | 재시작 | 보드·점수 초기화, 게임 재개 |
| 9 | CSS/JS 로드 | 스타일 적용됨, 네트워크 탭에서 404 없음 |

## GitHub Pages 배포 방법

### 1. 저장소 준비

```bash
git init
git add index.html style.css script.js README.md .gitignore
git commit -m "Prepare Tetris for GitHub Pages"
git branch -M main
git remote add origin https://github.com/<사용자명>/<저장소명>.git
git push -u origin main
```

### 2. Pages 활성화

1. GitHub 저장소 → **Settings** → **Pages**
2. **Source**: Deploy from a branch
3. **Branch**: `main` / **Folder**: `/ (root)`
4. **Save** 클릭

### 3. 배포 확인

몇 분 후 아래 주소로 접속합니다.

```
https://<GitHub사용자명>.github.io/<저장소명>/
```

위 **품질 점검 방법** 체크리스트를 다시 실행해 배포 환경에서도 동일하게 동작하는지 확인하세요.
